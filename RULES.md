# 기한 계산 규칙 명세 (RULES.md)

> 이 문서가 기한 계산 로직의 **유일한 진실의 원천**이다.
> builder는 이 명세에 없는 규칙을 임의로 추가·추정하지 않는다. 불명확하면 멈추고 질문한다.
> 모든 규칙은 2026-06 기준 공식 소스로 검증됨. 출처는 각 섹션에 명시.

---

## 0. 범위

- 대상: 한국 특허출원의 **의견제출통지서(OA) 대응 기한**만 다룬다.
- 제외: 해외 특허(미국/유럽/PCT), 상표, 디자인, 거절결정불복심판, 등록료 납부 기한. (PLAN.md Non-goals와 일치)

---

## 1. 기본 제출기한

| 항목 | 값 | 근거 |
|---|---|---|
| 기산점 | 의견제출통지서 **발송일** | 통지서 기재 기준 |
| 기본 제출기간 | **4개월** (2025-07-11 시행규칙 개정으로 2개월 → 4개월 확대) | 특허청 공고 (2025.7) |
| 구버전 기간 | 2개월 (개정 전 발송분) | 동일 |

**구현 요구사항**: 기본기간은 사용자가 **[4개월(2025.7.11 이후 발송) / 2개월(그 이전 발송) / 직접 입력]** 중 선택. 기본값 4개월.
사유: 개정 적용 경계와 심사관 재량 기재를 코드가 판단하지 않고, 통지서에 실제 기재된 기간을 사용자가 고르게 한다. UI에 "통지서 좌측 상단의 제출기일/지정기간을 확인하세요" 안내 문구 필수.

## 2. 지정기간 연장

| 항목 | 값 | 근거 |
|---|---|---|
| 연장 단위 | 1개월 | 특허법 시행규칙 |
| 소명 없이 가능한 횟수 | **4회 (총 +4개월)** | 특허청 |
| 5회차 이상 | 정당한 사유 소명 필요 (계산기는 "소명 필요" 경고만 표시, 기한은 계산해서 보여줌) | 동일 |
| 일괄 신청 | 가능 (예: 한 번에 2개월). 수수료는 월차별 합산과 동일 | 특허청 관납료 안내 |

## 3. 연장 수수료 (특허료 등의 징수규칙 제2조 제1항 11의2)

연장기간의 **누적 월차** 기준 구간별 금액. 신청 횟수와 무관하게 누적 월차로 계산한다.

| 누적 연장기간 | 해당 월차 금액 |
|---|---|
| 1개월 이하분 (1개월차) | 20,000원 |
| 1개월 초과 ~ 2개월분 (2개월차) | 30,000원 |
| 2개월 초과 ~ 3개월분 (3개월차) | 60,000원 |
| 3개월 초과 ~ 4개월분 (4개월차) | 120,000원 |
| 4개월 초과분 | **1개월마다 240,000원** |

누적 합계 예시 (검증용):
- +1개월: 20,000원
- +2개월: 50,000원 (2+3)
- +3개월: 110,000원 (2+3+6)
- +4개월: 230,000원 (2+3+6+12)
- +5개월: 470,000원 (23만 + 24만)
- +6개월: 710,000원

공식 출처: 특허로(patent.go.kr) 관납료 안내 — "1회에 2개월 일괄신청 시 총 50,000원[(1개월차) 20,000 + (2개월차) 30,000]"

## 4. 기간 계산 방법 (특허법 제14조)

1. **초일불산입**: 기간의 첫날(발송일)은 산입하지 않는다. 기산일 = 발송일 + 1일.
2. **월 단위 계산은 역(曆)에 따른다**: 최후의 월에서 **기산일에 해당하는 날의 전일**에 만료.
   - 예: 발송일 3/10 → 기산일 3/11 → 4개월 기한 = 7/10
3. **해당일이 없는 경우 그 월의 말일**에 만료.
   - 예: 발송일 10/31 → 기산일 11/1 → ... / 발송일 1/30 → 기산일 1/31 → 1개월 기한 = 2월의 말일 (2/28 또는 윤년 2/29)
4. **말일이 공휴일·토요일·일요일·근로자의 날(5/1)이면 그 다음 첫 근무일**로 만료.

### 공휴일 데이터 구현 방침

- 고정 공휴일: 1/1, 3/1, 5/5, 6/6, 8/15, 10/3, 10/9, 12/25, 근로자의 날 5/1
- 음력 공휴일(설·추석 연휴, 부처님오신날) + 대체공휴일: **연도별 테이블 하드코딩 (2025~2027년)**, `holidays.ts`에 날짜 배열로.
- 임시공휴일은 미반영 — UI에 "임시공휴일은 반영되지 않습니다" 문구 필수.
- 2025~2027 음력 공휴일 정확 날짜는 builder가 구현 시 한국천문연구원/공공데이터 기준으로 채우되, 채운 테이블을 BUILD_LOG에 기록해 검증 가능하게 한다.

## 5. 출력 요구사항

OA 발송일 + 기본기간 입력 시 타임라인으로 표시:

| 시나리오 | 표시 항목 |
|---|---|
| 연장 0회 (기본) | 제출 기한일, D-day |
| +1 ~ +4개월 | 각 기한일, 누적 수수료, D-day |
| +5개월 이상 | 기한일 + 누적 수수료 + ⚠️ "5회차부터 정당한 사유 소명 필요" 배지 |

- 모든 기한일에 요일 표시. 공휴일 순연이 발생한 경우 "원래 만료일 → 순연된 날짜" 형태로 순연 사실 표시.
- **법적 면책 문구 (모든 결과 화면 하단 고정)**: "본 계산 결과는 참고용이며 법적 효력이 없습니다. 공식 기한은 특허청 통지서에 기재된 내용을 기준으로 하며, 임시공휴일 등은 반영되지 않을 수 있습니다."

## 6. 필수 테스트 케이스 (vitest)

| # | 발송일 | 기간 | 기대 만료일 | 검증 포인트 |
|---|---|---|---|---|
| T1 | 2026-03-10 | 4개월 | 2026-07-10 | 기본 케이스 (초일불산입 + 전일 만료) |
| T2 | 2026-01-30 | 1개월 | 2026-02-28 | 해당일 없음 → 월말 |

공휴일 순연 (`adjustForHolidays`):

## 7. 신뢰도 표시 (정직성)

- 섹션 1~3: 공식 소스(특허청 공고, 특허로 관납료, 징수규칙) 기반 — 높음
- 섹션 4: 특허법 제14조 (수십 년 안정 조문) — 높음
- 음력 공휴일 테이블: builder 구현 시 채움 — **배포 전 사람 눈으로 1회 대조 필요** (BUILD_LOG에 기록)
