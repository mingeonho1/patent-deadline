# [Next.js] 특허 OA 기한 계산기 만들기 — 초일불산입을 코드로 옮기다 월말에서 터진 버그

> 발행 전 메모: 아래 제목 후보·태그·SNS 요약은 글 끝에 붙어 있다. 스크린샷 1장 필요.

특허 의견제출통지서(OA)를 받으면 그날부터 기한 계산이 시작된다. 발송일 기준 4개월, 연장하면 1개월씩 최대 몇 번까지, 연장 수수료는 월차마다 다르게 누적되고, 말일이 공휴일이면 다음 근무일로 밀린다. 변리사 사무소 직원들은 이걸 매주 여러 건씩 달력과 수수료표를 번갈아 보며 수기로 계산한다 (틀리면 권리가 날아가는 계산을 손으로..).

## 만든 것

그래서 이틀 동안 **특허 기한 계산기**를 만들었다. OA 발송일과 기본기간(4개월/2개월/직접입력)을 넣으면 제출 기한과 연장 시나리오별(+1~+6개월) 기한·누적 수수료를 타임라인으로 보여준다. 수기로 5분 걸리던 계산이 입력 10초로 끝난다.

## 기술적 결정 1: 음력 공휴일은 API 대신 하드코딩

기한 말일이 공휴일이면 다음 근무일로 순연해야 해서 공휴일 데이터가 필수다. 선택지는 둘이었다.

- 공공데이터포털 특일 API 호출 ➜ 정확하지만 외부 의존성 + API 키 + 장애 지점이 하나 늘어난다
- 2025~2027년 테이블 하드코딩 ➜ 3년치 60개 날짜면 배열 하나로 끝난다

하드코딩을 골랐다. 이 제품은 "기한이 틀리면 존재 이유가 없는" 도구라서, 런타임에 API가 죽어서 순연 계산이 빠지는 경로 자체를 없애고 싶었다. 대신 채운 테이블 전체를 BUILD_LOG에 표로 남겨 사람이 눈으로 1회 대조하는 절차를 박았다. 실제로 이 대조에서 2027년 성탄절 대체공휴일(12/27)과 한글날 대체공휴일(10/11) 누락이 잡혔다 (대체공휴일 규칙은 설·추석은 일요일 겹침만, 어린이날·부처님오신날·성탄절은 토·일 겹침까지라 사람도 헷갈린다..).

## 기술적 결정 2: env 검증을 빌드 타임으로 끌어올리기

대기명단 저장을 Supabase로 붙이면서 환경변수 검증을 zod로 했는데, 처음엔 모듈 로드 시점 검증이라 `next build`는 통과했다. 문제는 Vercel에 env 등록을 빠뜨려도 배포가 성공하고, 첫 제출 시점에 서버 액션이 모듈 로드 단계에서 던지는 에러는 try/catch 바깥이라 폼이 로딩 상태에 영구 고착된다는 것 ➜ 이번 빌드의 유일한 검증 지표(대기명단)가 조용히 죽은 채 출시될 뻔했다.

```ts
// next.config.ts
import "./src/lib/env"; // 빌드 타임에 zod parse 실행

// src/lib/env.ts
const envSchema = z.object({
  SUPABASE_URL: z.string().url(),
  SUPABASE_SECRET_KEY: z.string().min(1),
});
export const env = envSchema.parse(process.env);
```

- next.config.ts가 env 모듈을 import하는 순간 검증이 실행된다
- env가 없거나 형식이 틀리면 배포 빌드가 변수명과 함께 즉시 실패한다 (키 값은 미노출)
- 부작용: CI에도 똑같이 적용돼서 빌드가 깨졌고, CI 워크플로에만 플레이스홀더 값을 주입해 해결했다

## 막혔던 지점: 초일불산입 + 전일만료, 월말 발송일에서 깨진다

특허법 제14조의 기간 계산은 "첫날은 빼고(초일불산입), 마지막 달에서 기산일 해당 날의 전날에 만료"다. 처음엔 "+1일 했다가 -1일 하니 상쇄된다"고 단순화해서 `발송일 + n개월`로 구현했고, 명세의 테스트 5개도 전부 통과했다.

리뷰 단계에서 반례가 나왔다. 발송일 2026-04-30 +1개월 ➜ 내 코드는 5/30, 법정 기한은 5/31. 발송일이 월말이면 기산일이 다음 달 1일로 넘어가는데(4/30 + 1일 = 5/1), 이러면 "해당 날의 전일"이 6/1의 전일인 5/31이 된다. 상쇄 가정이 월 경계에서만 깨지는 거라 테스트가 우연히 다 통과했던 것이다.. 기산일을 명시적으로 계산하는 코드로 갈아엎고 월말 케이스 3개를 회귀 테스트로 박았다. 날짜 계산 버그는 항상 월말·월초 경계에서 나온다는 걸 또 배웠다.

## 결과

- 이틀, 커밋 9개, src 1,396줄, vitest 44개 (법정 기한·수수료 명세 14개 포함)
- 리뷰어 검수 1라운드 FAIL 6건 ➜ 전부 수정 후 SHIP IT. 위의 월말 버그와 공휴일 누락이 여기서 잡혔다
- 대기명단 등록: 0건에서 시작 (이제 막 배포했으니 당연하지만, 다음 글에서 숫자로 보고하겠다)
- 아쉬움: 임시공휴일은 미반영이라 화면에 안내 문구로 때웠다. 자동화하려면 결국 API가 필요해서 다음 빌드 후보로 넘겼다
